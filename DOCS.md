## Введение

**Эльбрус** — это профессиональный портативный инструмент для диагностики, восстановления и экстренного ремонта операционной системы Windows. Всё в одном исполняемом файле, не требующем установки.

### Основные возможности

- **Диспетчер процессов** — управление запущенными процессами
- **Анализ автозагрузки** — обнаружение угроз в реестре
- **Полноценный редактор реестра** — полный доступ к системному реестру
- **Инструменты восстановления** — экстренное восстановление системы
- **Анализ CmdLine** — обнаружение команд до входа в систему

| Параметр | Минимальные требования |
|----------|----------------------|
| **Операционная система** | Windows 10 / Windows 11 (x64) |

| **Права** | Рекомендуется запуск от имени администратора |
| **Интернет** | Не требуется (автономная работа) |

## Установка и запуск

### Установка

**Эльбрус** не требует установки в традиционном смысле. Это портативное приложение.

1. **Скачайте файл** `Elbrus.exe` из раздела [Releases](https://github.com/ilabolAfk/Elbrus/releases)
2. **Создайте папку** (рекомендуется) для хранения файла
3. **Поместите файл** в созданную папку

### Настройка иконки (опционально)

Если вы хотите использовать собственную иконку для приложения:

1. Подготовьте файл иконки в формате `.ico`
2. Переименуйте его в `icon.ico`
3. Поместите в ту же папку, где находится `Elbrus.exe`

### Запуск

**Важно:** Для полноценной работы требуется запуск от имени администратора.

#### Способ 1: Через контекстное меню
1. Нажмите правой кнопкой мыши на `Elbrus.exe`
2. Выберите **"Запуск от имени администратора"**
3. Подтвердите действие в окне UAC

#### Способ 2: Через двойной клик
1. Дважды кликните по `Elbrus.exe`
2. Если программа не имеет прав администратора, она предложит перезапуститься с правами

### Что происходит при первом запуске

При первом запуске **Эльбрус**:
1. Проверяет наличие прав администратора
2. Если прав недостаточно — предлагает перезапуститься с правами
3. Загружает список процессов
4. Сканирует критические разделы реестра
5. Показывает главное окно с пятью вкладками

## Интерфейс

### Общий обзор

Главное окно **Эльбрус** состоит из:

- **Верхняя панель** — название, версия, статус прав, кнопка обновления
- **Вкладки** — переключение между режимами работы
- **Основная область** — содержимое активной вкладки
- **Статус-бар** — информация о текущей операции

### Элементы интерфейса

| Элемент | Описание |
|---------|----------|
| **Заголовок** | Название программы, версия, статус прав |
| **Кнопка "Обновить"** | Обновляет данные на всех вкладках |
| **Вкладки** | Переключение между режимами работы |
| **Статус-бар** | Информация о текущей операции |
| **Тёмная тема** | Интерфейс адаптирован для длительной работы |

### Статус прав

В правом верхнем углу отображается статус:

- **АДМИНИСТРАТОР** (зелёный) — все функции доступны
- **ОГРАНИЧЕННЫЙ ДОСТУП** (оранжевый) — часть функций недоступна

## Вкладка "Процессы"

### Описание

Вкладка **"Процессы"** отображает все запущенные процессы с возможностью их завершения.

### Колонки таблицы

| Колонка | Описание |
|---------|----------|
| **Идентификатор** | Уникальный номер процесса (PID) |
| **Имя процесса** | Название исполняемого файла |
| **Загрузка ЦП, %** | Процент использования процессора |
| **Память, МБ** | Объём используемой оперативной памяти |
| **Состояние** | Статус процесса (running, sleeping, zombie и т.д.) |

### Подсветка

- Процессы с загрузкой CPU > 50% подсвечиваются красным
- Это помогает быстро идентифицировать ресурсоёмкие процессы

### Функции

#### Обновление списка
Нажмите кнопку **"Обновить список"** для получения актуальных данных.

#### Завершение процесса
1. Выберите процесс в таблице (клик по строке)
2. Нажмите кнопку **"Завершить процесс"**
3. Подтвердите действие в диалоговом окне
4. Процесс будет завершён (сначала вежливо, затем принудительно)

### Пример использования

**Сценарий:** Зависло приложение, которое не закрывается.

1. Откройте вкладку **"Процессы"**
2. Найдите зависшее приложение по имени
3. Выберите его
4. Нажмите **"Завершить процесс"**
5. Подтвердите действие

## Вкладка "Автозагрузка"

### Описание

Вкладка **"Автозагрузка"** сканирует критические разделы реестра и показывает, какие программы запускаются автоматически при старте системы.

### Анализируемые разделы

| Раздел | Описание | Признак угрозы |
|--------|----------|----------------|
| **Run** | Стандартная автозагрузка | Любые программы |
| **RunOnce** | Одноразовая автозагрузка | Любые программы |
| **Winlogon** | Контроль входа в систему | Shell ≠ explorer.exe |
| **AppInit_DLLs** | Внедрение DLL | AppInit_DLLs не пуст |
| **CmdLine** | Команды до входа | CmdLine не пуст |

### Подсветка угроз

- Разделы с подозрительными значениями подсвечиваются красным
- Критические параметры (Shell, Userinit) проверяются автоматически
- В случае обнаружения угрозы появляется значок ⚠

### Функции

#### Сканирование реестра
Нажмите кнопку **"Сканировать реестр"** для обновления данных.

#### Восстановление Winlogon
Если обнаружены изменения в Winlogon:

1. Нажмите кнопку **"Восстановить Winlogon"**
2. Подтвердите действие
3. Параметры будут восстановлены до стандартных:
   - Shell = explorer.exe
   - Userinit = C:\Windows\System32\userinit.exe
4. **Важно:** После восстановления требуется перезагрузка

### Пример использования

**Сценарий:** После запуска антивируса подозревает вирус в автозагрузке.

1. Откройте вкладку **"Автозагрузка"**
2. Нажмите **"Сканировать реестр"**
3. Проверьте разделы на наличие подозрительных записей
4. Особое внимание уделите Winlogon и CmdLine
5. При обнаружении угрозы используйте **"Восстановить Winlogon"**

## Вкладка "Реестр"

### Описание

Вкладка **"Реестр"** предоставляет полноценный редактор системного реестра с возможностью навигации, создания, редактирования и удаления разделов и параметров.

### Навигация

#### Выбор куста
В выпадающем списке доступны:

| Куст | Описание |
|------|----------|
| **HKEY_CLASSES_ROOT** | Ассоциации файлов |
| **HKEY_CURRENT_USER** | Настройки текущего пользователя |
| **HKEY_LOCAL_MACHINE** | Системные настройки |
| **HKEY_USERS** | Настройки всех пользователей |
| **HKEY_CURRENT_CONFIG** | Текущая конфигурация |

#### Переход по пути
1. Введите путь в поле **"Путь"**
2. Нажмите кнопку **"Перейти"** или клавишу Enter
3. Отобразится содержимое раздела

#### Навигация по дереву
- Клик по папке открывает подраздел
- Клик по параметру показывает его значение

### Работа с разделами

#### Создать раздел
1. Нажмите кнопку **"Создать раздел"**
2. Введите имя нового раздела
3. Нажмите **"ОК"**

#### Удалить раздел
1. Выберите раздел (📁)
2. Нажмите кнопку **"Удалить"**
3. Подтвердите действие
4. **Важно:** Раздел должен быть пустым для удаления

### Работа с параметрами

#### Создать параметр
1. Нажмите кнопку **"Создать параметр"**
2. Заполните поля:
   - **Имя параметра** (оставьте пустым для значения по умолчанию)
   - **Тип данных**: `REG_SZ`, `REG_DWORD`, `REG_BINARY`, `REG_EXPAND_SZ`
   - **Значение** — содержимое параметра
3. Нажмите **"Создать"**

#### Редактировать параметр
1. Выберите параметр
2. Нажмите кнопку **"Редактировать"**
3. Измените значение
4. Нажмите **"ОК"**

#### Удалить параметр
1. Выберите параметр
2. Нажмите кнопку **"Удалить"**
3. Подтвердите действие

### Подсветка критических значений

Параметры Winlogon автоматически подсвечиваются:
- **Shell** — подсвечивается, если не `explorer.exe`
- **Userinit** — подсвечивается, если не `C:\Windows\System32\userinit.exe`

### Пример использования

**Сценарий:** Добавление программы в автозагрузку.

1. Откройте вкладку **"Реестр"**
2. Выберите куст: `HKEY_LOCAL_MACHINE`
3. Перейдите в раздел: `SOFTWARE\Microsoft\Windows\CurrentVersion\Run`
4. Нажмите **"Создать параметр"**
5. Имя: `MyApp`
6. Тип: `REG_SZ`
7. Значение: `C:\Program Files\MyApp\app.exe`
8. Нажмите **"Создать"**

## Вкладка "Восстановление"

### Описание

Вкладка **"Восстановление"** содержит инструменты для экстренного восстановления системы.

### Инструменты

#### 1. Ассоциации файлов

**Описание:** Восстанавливает стандартные ассоциации для файлов.

**Что восстанавливает:**
- `.exe` — исполняемые файлы
- `.lnk` — ярлыки
- `.bat` — пакетные файлы
- `.cmd` — командные файлы

**Использование:**
1. Нажмите кнопку **"Восстановить"**
2. Программа выполнит команды `assoc`
3. Ассоциации будут восстановлены

**Когда использовать:**
- После заражения вирусами, которые изменяют ассоциации
- При ошибках открытия файлов
- После неудачной установки программ

#### 2. Восстановление Winlogon

**Описание:** Восстанавливает параметры входа в систему.

**Что восстанавливает:**
- Shell = explorer.exe
- Userinit = C:\Windows\System32\userinit.exe

**Использование:**
1. Нажмите кнопку **"Восстановить"**
2. Подтвердите действие
3. Параметры будут записаны в реестр

**Важно:** После восстановления требуется перезагрузка!

**Когда использовать:**
- При заражении блокировщиками Windows
- Когда после входа пароля ничего не происходит
- Когда вместо рабочего стола появляется чёрный экран

#### 3. Очистка временных файлов

**Описание:** Удаляет временные файлы для освобождения места.

**Что очищает:**
- Папка `%TEMP%` (временные файлы пользователя)
- Папка `%TMP%` (временные файлы системы)
- `C:\Windows\Prefetch` (кэш предварительной загрузки)

**Использование:**
1. Нажмите кнопку **"Очистить"**
2. Подтвердите действие
3. Программа удалит все временные файлы

**Когда использовать:**
- При нехватке места на диске
- Для ускорения работы системы
- После удаления вирусов

#### 4. Удаление CmdLine

**Описание:** Удаляет подозрительные команды из реестра.

**Что удаляет:**
- Параметр CmdLine из раздела `Policies\System`

**Использование:**
1. Нажмите кнопку **"Удалить"**
2. Подтвердите действие
3. Параметр будет удалён из реестра

**Когда использовать:**
- При обнаружении угроз в анализе CmdLine
- При подозрении на программу-вымогатель
- При блокировке входа в систему

## Вкладка "CmdLine"

### Описание

Вкладка **"CmdLine"** анализирует команды, которые запускаются до входа в систему. Это важный инструмент для обнаружения программ-вымогателей и блокировщиков.

### Функции

#### Сканирование

1. Нажмите кнопку **"Сканировать"**
2. Программа проверяет параметр CmdLine в реестре
3. Отображает результат анализа в текстовом поле

#### Результаты анализа

**Статус: БЕЗОПАСНО**
- Параметр CmdLine отсутствует
- Подозрительных команд не обнаружено

**Статус: ОБНАРУЖЕНА УГРОЗА**
- Параметр CmdLine содержит команду
- Отображается значение параметра
- Выводится предупреждение с рекомендацией удалить

#### Обнаружение угроз

Программа автоматически обнаруживает подозрительные команды:
- Содержат ключевые слова: `virus`, `malware`, `trojan`, `hack`, `crypt`
- Используют расширения: `.bat`, `.ps1`
- Запускают неизвестные исполняемые файлы

#### Удаление угрозы

1. Если обнаружена угроза, нажмите кнопку **"Удалить угрозу"**
2. Подтвердите действие
3. Параметр CmdLine будет удалён из реестра

### Пример использования

**Сценарий:** После включения компьютера появляется чёрный экран с требованием оплаты.

1. Запустите **Эльбрус** через командную строку (Ctrl+Alt+Del → Диспетчер задач → Файл → Запустить новую задачу)
2. Откройте вкладку **"CmdLine"**
3. Нажмите **"Сканировать"**
4. Если обнаружена угроза — нажмите **"Удалить угрозу"**
5. Перезагрузите компьютер

## Часто задаваемые вопросы

### Общие вопросы

#### Вопрос: Зачем нужен Эльбрус, если есть стандартные средства Windows?

**Ответ:** Эльбрус объединяет несколько инструментов в одном компактном портативном приложении:
- Не требует установки
- Всегда под рукой (можно носить на флешке)
- Автоматически обнаруживает угрозы в автозагрузке
- Специализированные инструменты для экстренного восстановления

#### Вопрос: Безопасно ли использовать Эльбрус?

**Ответ:** Да, но с осторожностью:
- Все операции требуют подтверждения
- Программа не изменяет систему без вашего согласия
- Рекомендуется создавать точки восстановления перед серьёзными изменениями

#### Вопрос: Почему программа требует права администратора?

**Ответ:** Для работы с системным реестром и завершения процессов требуются повышенные права. Без них некоторые функции будут недоступны.

### Технические вопросы

#### Вопрос: Почему загрузка CPU показывает больше 100%?

**Ответ:** Это нормальное поведение для многоядерных процессоров. Windows показывает суммарную загрузку всех ядер. Например, для 12-ядерного процессора максимальное значение — 1200%.

#### Вопрос: Что означает "System Idle Process" с высоким значением?

**Ответ:** Это процесс, показывающий свободное время процессора. Высокое значение означает, что система не нагружена. Чем выше — тем лучше.

#### Вопрос: Можно ли восстановить удалённые параметры реестра?

**Ответ:** Нет, Эльбрус не создаёт резервные копии. Рекомендуется создавать точки восстановления системы перед редактированием реестра.

### Проблемы

#### Вопрос: Программа не запускается, пишет ошибку.

**Ответ:** Проверьте:
1. Запущена ли программа от имени администратора
2. Не блокирует ли антивирус файл
3. Соответствует ли версия Windows требованиям

#### Вопрос: Не завершается процесс.

**Ответ:** Возможные причины:
1. Процесс имеет системный приоритет
2. Запущены другие экземпляры программы
3. Недостаточно прав

#### Вопрос: Автозагрузка не отображается.

**Ответ:** Проверьте:
1. Есть ли права администратора
2. Не повреждён ли реестр
3. Используйте кнопку "Сканировать реестр" для обновления

## Устранение неисправностей

### Ошибка: "Программа запущена без прав администратора"

**Решение:**
1. Закройте программу
2. Нажмите правой кнопкой мыши на `Elbrus.exe`
3. Выберите "Запуск от имени администратора"
4. Подтвердите действие в окне UAC

### Ошибка: "Не удалось завершить процесс"

**Решение:**
1. Проверьте, что у вас есть права администратора
2. Убедитесь, что процесс не является системным
3. Попробуйте завершить процесс через стандартный диспетчер задач

### Ошибка: "Не удалось записать в реестр"

**Решение:**
1. Проверьте права доступа к разделу реестра
2. Убедитесь, что программа запущена от имени администратора
3. Проверьте, не повреждён ли реестр

### Ошибка: "Файл не найден"

**Решение:**
1. Проверьте, что файл `elbrus.py` существует
2. Проверьте путь к файлу
3. Убедитесь, что все необходимые файлы в одной папке

### Программа не запускается

**Решение:**
1. Проверьте, не блокирует ли антивирус
2. Добавьте `Elbrus.exe` в исключения антивируса

## Сборка из исходного кода

### Требования

- Python 3.8 или выше
- PyInstaller 6.0 или выше
- PyQt6
- psutil

### Установка зависимостей

```bash
pip install PyQt6 psutil pyinstaller
```

### Сборка

#### Способ 1: Через командную строку

```bash
pyinstaller --onefile --windowed --name "Elbrus" --icon="icon.ico" --hidden-import psutil --hidden-import PyQt6 elbrus.py
```

#### Способ 2: Через build.bat

Создайте файл `build.bat` с содержимым:

```batch
@echo off
echo Сборка Эльбрус v3.0...
pyinstaller --onefile --windowed --name "Elbrus" --icon="icon.ico" --hidden-import psutil --hidden-import PyQt6 elbrus.py
echo Готово!
pause
```

#### Способ 3: Через Python скрипт

Создайте файл `build.py`:

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-

import os
import subprocess
import shutil

def main():
    print("=" * 50)
    print("  СБОРКА ЭЛЬБРУС v3.0")
    print("=" * 50)
    
    # Очистка старых файлов
    for folder in ["build", "dist"]:
        if os.path.exists(folder):
            shutil.rmtree(folder)
    
    for file in os.listdir("."):
        if file.endswith(".spec"):
            os.remove(file)
    
    # Сборка
    cmd = [
        "pyinstaller",
        "--onefile",
        "--windowed",
        "--name", "Elbrus",
        "--hidden-import", "psutil",
        "--hidden-import", "PyQt6",
        "elbrus.py"
    ]
    
    if os.path.exists("icon.ico"):
        cmd.insert(cmd.index("--name"), "--icon=icon.ico")
    
    subprocess.run(cmd)
    
    print("=" * 50)
    print("  СБОРКА ЗАВЕРШЕНА")
    print("=" * 50)
    print("  Файл: dist/Elbrus.exe")

if __name__ == "__main__":
    main()
```

### Результат сборки

После успешной сборки вы получите файл `Elbrus.exe` в папке `dist`.
